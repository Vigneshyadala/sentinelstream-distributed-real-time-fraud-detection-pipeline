02 · Detection Engine

Three sliding-window rules, evaluated per event

No batch jobs, no cron. Every rule re-evaluates the instant a new transaction lands, against a rolling window of that user's recent history held in memory.

RULE 01
Velocity Burst
Flags a user firing an abnormal number of transactions in a short window — the signature of card-testing bots.
> 3 txns / 10s window
RULE 02
High-Amount Outlier
Any single transaction that clears an absolute dollar threshold is flagged for review regardless of user history.
amount > $10,000
RULE 03
Impossible Travel
Compares consecutive locations for the same user via haversine distance — flags speeds no commercial flight can achieve.
implied speed > 900 km/h
03 · System Design

Lambda architecture: speed layer + batch layer

Flagged alerts take the fast path into TimescaleDB for sub-second queries. Every raw event — fraud or not — also lands in a MinIO data lake as partitioned Parquet, for offline model training and audit trails.
